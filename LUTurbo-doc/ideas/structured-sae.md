# Idea: 结构化 SAE — 训练时引入分组约束，联合解决 B 和 C1

> 关联文档：[research-log.md](../research-log.md)（主文档）、[decision-tree.md](../decision-tree.md)（决策树）
> 涉及决策点：B1-constrained（结构约束 SAE）、C1c（分组路由）、A2b（条件分解）
> 状态：待验证
> 动机来源：C1c oracle 实验（全子库 90-100%N）揭示 SAE 激活缺乏聚类结构，根因是纯重构训练目标

## 1. 核心想法

### 问题

C1c 条件子库 oracle 实验（[实验文档](../experiments/20260316-activation-patterns.md) §7.4）显示：对 SAE 激活模式做 KMeans 聚类后，每个簇的子库大小都在 90-100%N。不同簇的 token 使用的基向量几乎完全重叠，无法实现有效的"分区"——条件分解（A2b）因此不可行。

### 根因分析

这**不是激活空间的固有限制，最可能的解释是当前训练目标没有显式鼓励分组结构**。当前 SAE 的训练损失由重构误差（FVU）和辅助死特征损失（auxk_loss）组成：

```
L = FVU + auxk_alpha * auxk_loss,  x̂ = TopK(Ex + b) · D
```

两项损失都不包含任何关于基向量共激活模式的信号。TopK 从 N 个基向量中全局选 K 个，训练过程中没有引导"某些基向量应该一起出现"或"不同组应该互斥"的约束。在这种训练目标下，基向量倾向于学成尽可能"通用"的表示——每个都能被各种 token 使用，最大化覆盖面。这是缺乏聚类结构的最自然的解释，但尚未被严格排除其他可能性（如激活空间本身就没有可利用的分组结构）。

### 核心 idea

**修改 SAE 训练方式，使基向量自然形成 G 个组，每个输入主要激活少数几个组内的基向量。** 这同时改善两个瓶颈：

- **B（基向量库）**：分组后每个组内的基向量更专精，可能提升组内重构质量
- **C1（选择算法）**：选择从全局 N 降级为"选组 O(G) + 组内搜索 O(N/G)"，总开销可降 G 倍

## 2. 三种具体方案

### 2.1 Group TopK（改激活函数）

**原理**：把 N 个基向量分成 G 组（每组 N/G 个），将全局 TopK 替换为两级选择：

```
原来：z = TopK(Ex + b, K)           全局从 N 中选 K

改为：
  1) 用独立路由器 R ∈ R^{G × h} 计算组级评分 score_g = R_g · x，选 top-g 个最活跃的组
  2) 在被选中的 g 个组内做 TopK，总共选 K 个
```

**注意**：组级评分必须用独立的小型路由器 R（G 行 × h 列），而非从组内编码器 E_g·x 聚合。如果 score_g 是从 E_g·x 聚合的，那本质上仍需计算全部 N 个 latent logits，复杂度不会降低。独立路由器是 O(h×G) 复杂度成立的前提。

**训练**：路由器 R 和组内编码器 E_g 联合训练。组选择可通过 straight-through estimator 或 Gumbel-Softmax 使其可微分。梯度自然引导同一组的基向量学习相关特征。

**推理选择开销**（使用独立路由器时）：
- 路由器评分：O(h × G)（G << N，极低）
- 组内搜索：O(h × g × N/G)
- 总计：O(h × (G + gN/G))
- 当 g=4, G=64, N=8192 时：O(h × (64 + 4×128)) = O(h × 576) vs 原来 O(h × 8192) → **约 14 倍加速**
- 但路由器引入了额外参数（G×h）和一次 matmul，实际加速取决于 G 和 g 的选择

**与 MoE 的关系**：本质上是 Mixture of Experts 用在 SAE 潜空间。每个"专家"是一组基向量。区别在于 MoE 通常各专家独立，这里各组共享同一个输入空间。

### 2.2 共激活正则化（加损失项）

**原理**：保持 TopK 不变，额外加正则项鼓励分组结构：

```
L = E[||x - x̂||²] + λ · L_group
```

预先将 N 个基向量划分为 G 组，L_group 鼓励同组基向量共激活、抑制跨组共激活。例如：

```
L_group = -Σ_{同组(i,j)} CoAct(i,j) + Σ_{跨组(i,j)} max(CoAct(i,j), 0)
```

其中 CoAct(i,j) 是 batch 内 i 和 j 同时激活的频率。

**优点**：最小架构改动，与现有 TopK 兼容，可以逐步增大 λ 观察 tradeoff。

**缺点**：间接约束——鼓励但不保证分组结构。λ 调参敏感。

### 2.3 分块对角编码器（改架构）

**原理**：约束编码器矩阵为分块对角结构：

```
原来：E ∈ R^{N × h}，稠密

改为：E = block_diag(E_1, E_2, ..., E_G)
     每个 E_g ∈ R^{(N/G) × (h/G)}
```

每组编码器只看输入 x 的 h/G 个维度。

**选择开销**：每组独立计算，总开销 = G × (N/G) × (h/G) = **N×h/G**，直接缩小 G 倍。

**优点**：选择问题被架构直接解决——各组独立并行，天然适合 CPU 多核。

**缺点**：约束最强。如果输入 x 的各子空间之间有强相关性（跨子空间特征），分块对角无法捕获。可能需要残差连接或小型混合层缓解。

## 3. 预期收益与量化目标

基于 C1c oracle 数据的目标：

| 指标 | 当前（标准 SAE） | 目标（结构化 SAE） |
|------|:---:|:---:|
| 全子库/N | 90-100% | <50%（理想 <25%） |
| 25%N 截断 recall | 57-84% | >90% |
| C1 选择开销 | O(h × N) | O(h × N/G) |
| 重构 MSE 相对标准 SAE | 基线 | 允许上升 <20% |

**核心判据**：如果结构化 SAE 的重构 MSE 增加 <20%，且全子库缩减到 <50%N（使 25%N 截断 recall >90%），则方案整体收益为正——C1 选择开销的大幅降低远超 MSE 的小幅上升。

## 4. 核心 tradeoff

```
更强的分组结构 → C1 选择更容易（搜索空间从 N 降到 N/G）
更强的分组结构 → 重构质量可能下降（基向量表达自由度被限制）
```

当前 SAE 处于 tradeoff 曲线的一个极端：重构最优、选择最难。我们需要找到 Pareto 前沿上的另一个点：稍微牺牲重构质量，换取选择开销大幅下降。

**参考**：o_proj 天然具有更强的频率不均匀性（Gini 0.58-0.88），hotset recall 高（72-94%），暗示 o_proj 的激活空间可能更容易通过训练引入分组。MLP/QKV（Gini 0.35-0.56）均匀性更强，分组约束对 MLP/QKV 可能需要更强的正则化。

## 5. 与决策树的关系

本 idea 位于决策树 B1-constrained 节点。它创建一条新的耦合路径：

```
B1-constrained(结构化SAE训练)
→ 激活具备分组结构
→ C1c(条件子库) 重新可行
→ A2b(条件分解) 成为可能
→ 每个子库更小更专精
→ C1 搜索空间缩小 + D 精度提升
```

这是杠杆效应最大的路径之一：改变 B 的训练方式，同时解锁 C1c 和 A2b 两个被阻塞的节点。

## 6. 验证计划

### Phase 1：Group TopK 原型（优先级最高）

在 Qwen3-0.6B 上训练 Group TopK SAE：
- 参数：G=16/32/64, g=2/4/8（每个输入选 g 个组）
- 对比基线：同 N、同 K 的标准 SAE
- 观测：(1) 重构 MSE 相对变化；(2) 全子库大小/N；(3) 25%N 截断 recall
- **go/no-go**：MSE 增加 <20% 且全子库 <50%N → 进入 Phase 2

### Phase 2：三方案对比

在同一模型/数据上对比 Group TopK、共激活正则、分块对角三种方案：
- 统一评估框架：MSE vs 子库压缩率 Pareto 曲线
- 确定最优方案及参数

### Phase 3：C1c 复测

用结构化 SAE 的激活数据重跑 C1c oracle pipeline：
- 验证子库是否真正缩小
- 测量 oracle routing recall 是否达到 >90%@N/G

## 7. 风险

1. **重构质量大幅下降**：分组约束可能让 MSE 上升远超 20%，导致 p（补偿比例）大幅增加，抵消 C1 的收益。
2. **分组结构不稳定**：训练出的分组可能对数据分布敏感，换数据集后分组失效。
3. **超参增多**：G、g、λ 的选择空间大，调参成本高。建议先固定 G=32 做 sweep。
4. **o_proj vs MLP/QKV 差异**：分组约束对天然有结构的 o_proj 可能效果好，对更均匀的 MLP/QKV 可能不够。
