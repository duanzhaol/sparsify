# Idea: SAE 改进 — 降低 K 需求与引入结构化激活

> 关联文档：[research-log.md](../research-log.md)（主文档）、[decision-tree.md](../decision-tree.md)（决策树）
> 涉及决策点：B（基向量库构建）、C1（选择算法）、C2（系数计算）
> 状态：待验证（诊断实验未开始）
> 本文档合并了原 `structured-sae.md` 的内容，作为 SAE 训练改进的统一方案

## 1. 问题与目标

### 1.1 当前状况

- **K=128（h=1024）** 才能保证重构精度，这导致查表访存 128n、选择开销高
- 激活模式实验（[experiments/20260316-activation-patterns.md](../experiments/20260316-activation-patterns.md)）揭示的结构性问题：
  - SAE 激活缺乏聚类结构（C1c 全子库 90-100%N）
  - 相邻 token 重叠率低（C1e 仅 20-37%）
  - MLP/QKV 频率分布均匀（Gini 0.35-0.56），C1h+C1i 在 16%N 候选下仅 74% recall
- CG 实验证明系数不是瓶颈（MSE 仅降 10-17%），**主矛盾在选择和字典质量**

### 1.2 两个互补目标

**Goal A：降低 K 需求**（从 128 降到 32-64）
- 查表访存降 2-4x
- C1 选择变简单（从 N 中选 32 个 vs 128 个，ANNS 可能重新可行）
- CG 开销降 2-4x，可能重回主线
- 补偿 p 降低（更好重构 → 更小 τ 下仍有低 p）

**Goal B：引入分组结构**（使 C1c/A2b 可行）
- 选择从全局 N 降级为"选组 + 组内搜索"
- 搜索空间从 N 缩小到 N/G
- MLP/QKV 的 C1 问题可能被根本解决

两者的关系：
- **可能协同**：更好的字典 → K 更小 → 每组内选更少 → C1 更简单
- **可能冲突**：分组约束 → 表达力受限 → 需要更大 K
- 需要实验判断 tradeoff，最终在 Pareto 前沿上选点

## 2. 诊断阶段（在尝试任何改进之前）

在尝试改进之前，必须先理解当前 SAE 的失败模式。以下三个诊断实验成本极低（用现有 checkpoint），但信息价值极高。

### 2.1 Oracle K-重构曲线（核心诊断）

**目的**：分离"字典质量"和"选择质量"两个因素，确定 K=32 的天花板在哪里。

**方法**：用现有 K=128 SAE 的解码器字典 D（N 个基向量不变），对每个测试样本 x：

```
对 K' = 16, 32, 64, 96, 128：
  (a) 编码器选择：标准 TopK(Ex+b)，取前 K' 个，用内积系数重构 → FVU_encoder(K')
  (b) OMP 选择：用 OMP 贪心选 K' 个基向量（每步选择 + 重优化系数）→ FVU_omp(K')
  (c) PCA 基线：用训练数据的前 K' 个主成分方向投影 → FVU_pca(K')
```

**实现说明**：(b) 推荐用 OMP（Orthogonal Matching Pursuit）而非朴素贪心。OMP 在每步选择后用 lstsq 重新优化已选基向量的系数再计算残差，选择质量略优于朴素贪心（残差方向更准确），且有成熟实现（`sklearn.linear_model.OrthogonalMatchingPursuit`）。OMP 本身已包含每步的系数优化，因此原来的 (b) 贪心 + (c) lstsq 合并为一步。

**三条曲线的解读**：

| 比较 | 含义 |
|------|------|
| (c) PCA 曲线 | 激活空间的内在维度。如果 PCA K'=32 就能解释 >95% 方差 → K=32 理论可行 |
| (b) vs (c) | 当前字典比 PCA 好多少。过完备字典理论上应该优于 PCA |
| (a) vs (b) | 编码器选择的质量损失。gap 大 → 选择方法有大改善空间 |

**关键判定**：

| OMP(32) vs encoder(128) | 结论 | 后续方向 |
|:---:|------|------|
| OMP(32) ≈ encoder(128) | 字典够好，问题在选择 | 优先改选择方法（C1）、Gated SAE 解耦选择 |
| OMP(32) 明显差于 encoder(128) | 字典不适合小 K | 需要改字典：重训练、换架构、换 N/K |
| OMP(32) 差且 PCA(32) 也差 | 全局低秩和当前字典下小 K 都不乐观 | K=32 风险高，优先转向 K=64 或多级分解 |

### 2.2 死特征审计

**目的**：检查当前训练是否浪费了字典容量。

**方法**：
- 统计每个基向量在测试数据上的激活频率
- 计算死特征比例（激活频率 = 0 或极低的特征数 / N）
- 确认训练时 `auxk_alpha` 的实际设置值

**为什么重要**：代码默认 `auxk_alpha = 0.0`（死特征处理关闭）。如果实际训练未开启，N=8192 中可能有大量死特征，有效字典远小于 8h。这是最容易修复的问题——仅调一个超参就可能显著提升重构质量。

### 2.3 PCA 维度分析

**目的**：提供全局低秩可压缩性的基线参考。

**方法**：
- 对各层各算子的激活数据做 PCA
- 记录前 K' = 16, 32, 64, 128, 256 个主成分的累积解释方差比例
- 逐层逐算子绘制曲线

**重要澄清**：PCA 给出的是"固定 K' 维线性子空间"的最优误差，对所有样本用同一组基。而 OMP/SAE 是"每个样本自适应选 K' 个原子"的稀疏表示（union of subspaces），表达力远强于单个子空间。因此：
- PCA@32 解释方差低 **不意味着** OMP@32 或 SAE@32 也差
- 正确的关系是：PCA@K ≤ OMP@K ≤ 理论最优稀疏编码@K
- PCA 只是全局低秩基线，不是稀疏编码的上界
- 真正决定 K=32 可行性的是 OMP@32 的结果（§2.1）

### 2.4 Latent 分组 Oracle（结构化 SAE 预诊断）

**目的**：在投入 Group TopK 训练之前，先用现有 SAE 的 latent 结构判断"分组是否有潜力"。

**背景**：C1c 实验（token 聚类 → 子库）失败了（全子库 90-100%N）。但 C1c 测的是"不同类型 token 是否用不同基向量"，而 Group TopK 问的是"基向量能否分成几组，使得每个输入主要需要少数几组"——这是不同的问题。

**方法**：
- 对现有 SAE 的 N 个 latent 做离线分组：
  - 方案 1：按 decoder 余弦相似度聚类（基于基向量方向）
  - 方案 2：按 PMI 共激活图谱系聚类（基于使用模式，复用 C1i 的 PMI 表）
- 分成 G = {8, 16, 32, 64} 组
- 对每个测试样本的真实 top-K，测量"top-g 组覆盖了多少 top-K"：
  - group_recall@g：选最活跃的 g 个组，组内 latent 并集 ∩ 真实 top-K / K
  - 扫描 g = {2, 4, 8, 16}

**关键判定**：
- 对比 **随机分组基线**：同样 G、g 下随机分配 latent 到组的 group_recall（期望值 ≈ g/G）。真实分组 recall 相对随机基线的提升倍数是判断"是否有结构"的核心指标
- group_recall@g=4 显著高于随机基线且 > 85% → Group TopK 很有潜力，值得投入训练
- group_recall@g=4 接近随机基线或 < 60% → 当前 latent 无自然分组结构。Group TopK 仍可能通过训练强制形成分组，但风险更高
- 注意：即使 oracle 差，Group TopK 训练可能通过梯度信号迫使分组形成——这个 oracle 是充分条件（好 → 必做），不是必要条件（差 → 不一定不做，但风险高）

## 3. Goal A 的改进方向：降低 K 需求

以下方案按"预期信息价值 / 实现成本"排序。

### 3.1 训练超参优化（低垂果实）

#### 3.1.1 auxk_alpha × dead_feature_threshold 联合调优

如果诊断发现死特征比例高（>30%），这是第一优先修复项。

**方案**：联合 sweep `auxk_alpha` ∈ {0, 1/64, 1/32, 1/16} × `dead_feature_threshold` ∈ {1_000_000, 10_000_000}，固定 K=128，对比 FVU。

**为什么需要联合 sweep**：代码中死特征机制是过了 `dead_feature_threshold` 个 token 不激活才生效（默认 1000 万 token）。如果训练总 token 数不足以触发该阈值，`auxk_alpha` 再大也没有效果。两者必须同时考虑。

**预期**：Gao et al. 2024 推荐 auxk_alpha=1/32。开启 auxk 应能激活更多特征 → 有效字典更大 → 重构质量提升。

#### 3.1.2 优化器对比

当前默认 SignSGD（`signum`），这在 SAE 文献中不常见——sign 操作丢弃梯度幅度信息，可能不利于稀疏结构的精细优化。

**方案**：对比 signum / adam / muon，固定其他参数，比较 FVU 收敛曲线。当前主干 Trainer 硬编码 SignSGD，切换优化器需要修改 `trainer.py`（requires infra change）。

#### 3.1.3 Hadamard 旋转

代码已实现 `use_hadamard` 选项。Hadamard 旋转将激活的离群维度能量分散到所有维度，使分布更均匀。

**假设**：如果少数维度有极大值（outlier dimensions），SAE 需要花费大量特征来表示这些维度。旋转后分布更均匀，SAE 学习更容易。

**方案**：on/off 对比，观察 FVU@K=128 和 FVU@K=32。

### 3.2 Gated SAE（选择/系数解耦）

**原理**：将编码器分为两个独立网络——gate 网络决定选哪些特征，magnitude 网络决定系数大小：

```
当前：  z = TopK(ReLU(W_enc · x + b_enc))      编码器同时做选择和系数

Gated：
  gate:      g = TopK(σ(W_gate · x + b_gate))   二值选择（哪些特征）
  magnitude: m = W_mag · x + b_mag               连续系数（多大权重）
  output:    z = g ⊙ m
```

**为什么可能降低 K 需求**：
- 当前 TopK(ReLU(Ex+b)) 让编码器同时做分类和回归，两个目标互相干扰
- 解耦后，gate 可以专注于选最"有用"的特征，magnitude 专注于给最优系数
- 逻辑与 CG 实验一致——CG 证明解耦系数能提升精度。Gated SAE 在训练阶段就实现了这种解耦

**对 LUTurbo 的额外价值**：
- gate 网络可以比编码器更小（只需二值选择，不需要精确系数）
- gate 网络可能更容易被低秩近似或结构化（与 C1f 协同）

**实现难度**：中。需要修改 `sparse_coder.py` 的编码器部分，增加 gate 分支。

### 3.3 JumpReLU SAE（可学习阈值激活）

**原理**：将 TopK 替换为每特征独立的可学习阈值：

```
当前：  z = TopK(ReLU(Ex + b), K)              固定 K，全局竞争

JumpReLU：
  z_i = max(0, (Ex + b)_i) · 1((Ex + b)_i > θ_i)   每个特征 i 有独立阈值 θ_i
```

K 不再固定，而是由输入决定——简单输入可能只激活 30 个特征，复杂输入可能激活 200 个。训练时 θ_i 通过 straight-through estimator（STE）学习。

**为什么可能学到更好的字典**：
- TopK 强制每个输入恰好用 K 个特征：简单输入被迫选入无用特征（引入噪声梯度），复杂输入又不够用（梯度信号截断）
- JumpReLU 让每个输入用"恰好需要的量"，训练信号更干净 → 字典质量可能更高
- DeepMind 2024 的实验显示 JumpReLU SAE 在同等稀疏度下重构质量优于 TopK SAE

**对 LUTurbo 的特殊考量**：
- **核心矛盾**：LUTurbo 需要可预测的 K 来保证查表访存确定，但 JumpReLU 的 K 随输入变化
- **可能的折中策略**：
  - **训练时 JumpReLU，部署时 TopK**：用 JumpReLU 训练获得更好的字典，部署时切回 TopK(K=32)。风险是 train-deploy mismatch——字典为变长 K 优化，固定 K 截断时可能不是最优选择
  - **JumpReLU + 硬上界**：`z_i = max(0, (Ex+b)_i) · 1((Ex+b)_i > θ_i)`，但额外限制总激活数 ≤ K_max。超出时取 top-K_max。这保留了 JumpReLU 对简单输入的"少用特征"优势，同时给出延迟上界
  - **统计控制**：通过调整全局 θ 偏移使平均 K ≈ 目标值，接受一定的 K 波动。如果 K 的 P99 < 2×均值，工程上可能可接受

**需要关注的问题**：
- STE 的训练稳定性：阶跃函数梯度近似可能导致训练不稳定
- K 的实际分布：均值、方差、P99——如果方差太大，延迟不可预测
- Train-deploy mismatch 的量化影响：JumpReLU 训练的字典在 TopK=32 部署时 FVU 损失多少？

**实现难度**：中。需要修改激活函数和训练循环（STE 梯度），但架构改动不大。

### 3.4 Matryoshka 训练（多 K 联合优化）

**原理**：在同一次训练中同时优化多个 K 值的重构质量：

```
z = TopK(ReLU(Ex + b), K_max)    # 先算 K_max 个激活
z 按激活值降序排列

L = Σ_{K' ∈ {32, 64, 128}} w_{K'} · FVU(x, z[:K'])
```

**为什么可能降低 K 需求**：
- 强制 SAE 学习一种"重要性排序"——最强的 32 个激活就已经给出不错的重构
- 标准训练只优化 K=128 时的 FVU，不关心子集质量；Matryoshka 显式要求子集也好
- 部署时可以按 K=32 截断，实现优雅降级

**需要关注的问题**：
- 权重 w_{K'} 的设置：较大的 K' 给较小权重？还是均等？
- 是否会牺牲 K=128 时的最优性（如果只关心小 K，可以接受）
- 与标准 TopK 训练是否有根本冲突（可能需要排序稳定的 TopK 实现）

**实现难度**：低。只需修改损失函数，不改架构。

### 3.5 正交性约束

**原理**：鼓励解码器基向量之间更正交：

```
L = FVU + λ_orth · ||D_S^T D_S - I||_F^2
```

其中 D_S 是当前 batch 中被激活的基向量子集（不是全局 D^T D，那太大了）。

**为什么可能降低 K 需求**：
- 如果基向量高度相关，多个基向量覆盖同一方向是浪费
- 更正交 → 每个基向量贡献独立信息 → K 个特征覆盖 K 个独立方向
- 同时让 CG 收敛更快（Gram 矩阵条件数更好）

**与 Goal B 的协同**：正交基向量更容易形成自然分组（正交的子集之间天然低耦合）。

**实现难度**：低。在训练循环中加一项正则损失。

### 3.6 K 的课程学习

**原理**：训练分阶段，逐步减小 K：

```
Phase 1: K=256, 训练到收敛（字典学会覆盖整个空间）
Phase 2: K=128, 微调（字典开始压缩、合并冗余特征）
Phase 3: K=64/32, 微调（字典进一步专精）
```

**假设**：大 K 阶段让字典先获得全局覆盖能力，小 K 阶段迫使字典丢弃冗余、保留最有信息的特征。

**需要关注的问题**：
- 减 K 时是否重新初始化编码器？还是继续微调？
- 每阶段需要多少训练步数？
- 与 Matryoshka 的关系：课程学习是时间维度的渐进，Matryoshka 是损失函数维度的同时优化

**实现难度**：低。只需修改训练脚本的 K 调度。

### 3.7 N/K 参数空间探索

当前 N=8h 是"精度下界"，但这个结论是在 K=128 下得出的。N 和 K 之间可能存在非平凡的交互：

- **更大的 N + 更小的 K**：更多候选基向量 → 更精细的字典 → 可能用更少 K 达到同样精度。但选择空间变大。
- **更小的 N + 更小的 K**：字典更小，选择更容易，但表达力可能不够。

**需要的实验**：在 (N, K) = {4h, 8h, 16h} × {32, 64, 128} 的网格上测 FVU，找 Pareto 前沿。

### 3.8 Tiled SAE 探索

代码已实现 `TiledSparseCoder`——将 h 维输入分成 T 个 tile，每个 tile 独立编码：

```
x ∈ R^h → T 个 tile，每个 R^{h/T}
每个 tile: k_per_tile = K/T 个特征，字典大小 N/T
```

**为什么可能有效**：
- 每个 tile 维度更小（h/T），用更少特征就能覆盖
- 搜索空间从 N 降到 N/T（天然解决 C1）
- 但丢失 tile 间交互信息

已有 `global_topk` 和 `input_mixing` 选项可能缓解信息丢失。

**与 2.3 分块对角编码器的关系**：Tiling 是分块对角的一种已有实现，可以直接用来测试分块对角思路的可行性。

### 3.9 残差/多级 SAE（B1-residual）

**原理**：将重构分为两级——第一级捕捉主导结构，第二级修正残差：

```
Level 1: z1 = TopK(E1·x + b1, K1),   x̂1 = D1[S1]^T · α1      K1 个特征，粗重构
Level 2: r = x - x̂1
         z2 = TopK(E2·r + b2, K2),    r̂ = D2[S2]^T · α2       K2 个特征，残差修正
Total:   x̂ = x̂1 + r̂,                K_total = K1 + K2
```

**为什么可能降低有效 K / 改善 C1**：

与其他方案不同，残差 SAE 的独特优势不在于减小总 K，而在于**每级的选择问题结构不同**：
- **Level 1（粗）**：捕捉主导模式。可能只需少量基向量（K1=16-32），且激活模式高度集中——热集可能直接覆盖大部分 Level 1 选择 → C1h 直接解决
- **Level 2（细）**：在残差上操作。残差的统计特性与原始激活不同——可能更低维、更均匀、或有不同的频率分布。Level 2 可能需要不同的 C1 策略，但搜索空间（N2）可以更小

**类比**：类似 JPEG 的 DC + AC 分离——粗和细在不同尺度上操作，各自用更少的"基"就能表示好。

**训练方式**：
- **级联训练**：先训练 Level 1 到收敛，冻结后训练 Level 2。简单但 Level 2 受限于 Level 1 的残差分布
- **联合训练**：两级端到端训练，总损失 = ||x - x̂1 - r̂||²。更优但训练复杂度高

**需要关注的问题**：
- 总 K = K1+K2 vs 单级 K：两级 (K1=32, K2=32) 的 FVU 是否优于单级 K=64？
- 两轮 C1 选择的总开销：即使每轮更快，两轮的延迟叠加可能更大
- Level 2 的字典大小 N2：残差是否足够低维，使得 N2 << N1？
- 查表开销：两级各自的查找表需要分别存储（P1 = D1·W, P2 = D2·W），存储量翻倍

**实现难度**：中-高。需要修改训练循环（两级前向/反向传播），新增第二级 SAE，修改推理 pipeline。

## 4. Goal B 的改进方向：引入分组结构

### 4.1 动机

C1c 实验显示全子库 90-100%N，原因是训练目标（FVU + auxk_loss）不包含分组信号。基向量学成了"通用表示"——任何 token 都可能用到任何基向量。

**核心 idea**：修改训练方式，使基向量自然形成 G 个组，每个输入主要激活少数几个组内的基向量。选择从全局 N 降级为"选组 O(G) + 组内搜索 O(N/G)"。

### 4.2 Group TopK（改激活函数）

将全局 TopK 替换为两级选择：

```
1) 独立路由器 R ∈ R^{G × h} 计算组级评分 score_g = R_g · x，选 top-g 个组
2) 取选中 g 个组的 latent 并集（大小 g × N/G），在该并集上做全局 top-K
```

**注意**：不要强制 k_per_group = K/g（每组固定配额）。这样会把配额浪费给弱组。正确做法是在 g 个组的并集上做一次全局 top-K，让强组自然贡献更多特征。

**关键**：组级评分必须用独立路由器 R（G×h），不能从组内编码器聚合（否则仍需计算全部 N 个 logits）。

**推理选择开销**：O(h × (G + g×N/G))。当 g=4, G=64, N=8192 时 ≈ O(h × 576) vs 原来 O(h × 8192)，约 14x 加速。

**与 MoE 的关系**：本质上是 MoE 用在 SAE 潜空间，每个"专家"是一组基向量。

### 4.3 共激活正则化（加损失项）

保持 TopK 不变，额外加正则项鼓励分组：

```
L = FVU + λ · L_group

L_group = -Σ_{同组(i,j)} CoAct(i,j) + Σ_{跨组(i,j)} max(CoAct(i,j), 0)
```

**优点**：最小架构改动，可逐步增大 λ 观察 tradeoff。
**缺点**：间接约束，不保证分组结构。需要预先划分组（可用 K-means 初始化）。

### 4.4 分块对角编码器（改架构）

约束编码器为分块对角：

```
E = block_diag(E_1, ..., E_G),  E_g ∈ R^{(N/G) × (h/G)}
```

选择开销：N×h/G，直接缩小 G 倍。

**缺点**：约束最强。跨子空间特征无法捕获。可通过 Hadamard 旋转预处理缓解（先把相关性分散到各子空间）。

**注意**：这与 §3.8 的 TiledSparseCoder 本质相同——可用已有 Tiling 实现直接验证。

### 4.5 预期收益

| 指标 | 当前（标准 SAE） | 目标（结构化 SAE） |
|------|:---:|:---:|
| 全子库/N | 90-100% | <50%（理想 <25%） |
| 25%N 截断 recall | 57-84% | >90% |
| C1 选择开销 | O(h × N) | O(h × N/G) |
| 重构 MSE 相对标准 SAE | 基线 | 允许上升 <20% |

## 5. Goal A + B 联合方案

### 5.1 Group TopK + Matryoshka（最有希望的联合方案）

```
训练损失：
  z = GroupTopK(x, R, {E_g}, G, g, K_max)
  L = Σ_{K' ∈ {32, 64, K_max}} w_{K'} · FVU(x, z[:K'])
```

- Group TopK 引入分组 → 选择降级为"选组 + 组内选"
- Matryoshka 保证 Top-32 子集已有基本精度 → 可以选更少特征
- 两者在训练损失中自然叠加，不冲突

### 5.2 Gated SAE + 正交性约束

```
训练损失：
  g = TopK(σ(W_gate · x + b_gate))
  m = W_mag · x + b_mag
  z = g ⊙ m
  L = FVU + λ_orth · ||D_S^T D_S - I||_F^2
```

- Gated 解耦选择/系数 → 更好的特征利用
- 正交约束 → 每个特征独立贡献 → 更少 K 就够用
- 正交基向量天然形成低耦合子集 → 有利于后续分组

### 5.3 面向重构的字典学习（根本性重新定位）

标准 SAE 文献（Anthropic、Gao et al.）优化目标是可解释性（单语义特征）。LUTurbo 不关心可解释性，只关心重构效率。这意味着：

- 特征可以是多语义的（只要有利于重构）
- 可以借鉴字典学习（K-SVD）和压缩感知文献
- End-to-end 优化目标：直接优化"K=32 时的重构质量"

这是最激进的方向，实现成本高，但如果 Goal A 的其他方案都无法将 K 降到 32，可能需要走这条路。当前作为备选保留。

## 7. 与决策树的耦合

### 7.1 对 C1 的影响

K 降低直接改变 C1 可行性格局：

| C1 方案 | K=128 | K=32 |
|---------|-------|------|
| ANNS (C1b) | 不可行 | 可能重新可行 |
| 编码器近似 (C1f) | 不可行（K×h 太大） | 开销降 4x，可能进入可行区间 |
| C1h+C1i 组合 | o_proj 88%，MLP/QKV 74% | 选更少 → recall 可能更高 |

分组结构引入（Goal B）同时解锁 C1c 和 A2b：

```
B1-constrained → 激活有分组结构 → C1c 子库缩小 → A2b 条件分解可行
```

### 7.2 对 C2 的影响

- CG 开销 O(t·K·h)：K 从 128→32，开销降 4x → CG 可能重回主线
- 正交约束（§3.4）改善条件数 → CG 收敛更快
- Gated SAE 的 magnitude 分支可能已经给出接近最优的系数 → 不再需要 CG

### 7.3 对 D 的影响

- 更好的重构 → 更低的 p → 补偿访存减少
- 如果 K=32 + 改进 SAE 让 p@τ=0.3 ≈ 20%：
  - 查表：32n（vs 128n）
  - 补偿：0.2·h·n
  - 总计约 0.22·h·n + 32n ≈ 0.25·h·n（4x 加速）

## 8. 风险

1. **K=32 风险高**：如果 OMP@32 和 OMP@64 都远差于 encoder@128，说明当前字典和全局低秩结构都不支持小 K。注意 PCA 差不直接意味着稀疏编码也差（稀疏编码是 union of subspaces，表达力远强于 PCA），真正的判据是 OMP 曲线。Phase 0a 的 OMP 分析可以量化这个风险。

2. **分组约束导致重构质量大幅下降**：Group TopK 可能让 MSE 上升远超 20%，p 大幅增加，抵消 C1 收益。需要在 Phase 2d 中仔细监控 FVU vs 子库压缩的 tradeoff 曲线。

3. **Goal A 和 Goal B 冲突**：分组约束减小表达力 → 需要更大 K。如果出现这种情况，需要在 Pareto 前沿上选择：(K, G) 的某个折中点可能比 (K=32, G=1) 或 (K=128, G=64) 都更好。

4. **训练成本高**：多个架构变体 × 多个 (N, K, G) 配置 = 大量训练实验。建议先在小模型（Qwen3-0.6B）上快速筛选，只把最优方案放到大模型上验证。

5. **算子异质性**：o_proj 天然有频率集中性（Gini 0.58-0.88），改进方案可能对 o_proj 效果好但对 MLP/QKV 不够。可能需要不同算子使用不同 SAE 配置。
