# LUTurbo 决策树与耦合关系

> 本文档是 `research-log.md` 的辅助文档。
> 用树结构展示所有决策点和可选方案，用耦合矩阵展示跨分支的约束关系。
> "当前路径"标记了目前选定的方案组合。

## 1. 假设链（从根到叶）

```
A0: LLM 推理是 memory-bound（访存 ≈ 性能）
└─ A1: 用查表替代矩阵乘法可以降低访存
    └─ A2: 分解方式
        ├─ A2a: 全局基向量线性分解（所有输入共享一个基向量库）← 当前
        └─ A2b: 条件分解（不同输入区域用不同的基向量库）
```

## 2. 决策树

A2 之下分出三个独立的子问题，每个子问题有多个可选方案。

### B: 基向量库怎么构建？

决定基向量的质量（覆盖性、稀疏表示能力）。

| ID | 方案 | 状态 | 备注 |
|----|------|------|------|
| B1 | 常规 SAE（稠密编码器 + TopK + 解码器） | ← 当前 | 精度不够理想 |
| B1-gated | Gated SAE | 待验证 | 门控与系数计算解耦，可能精度更高 |
| B1-jump | JumpReLU SAE | 待验证 | 带阈值 ReLU，稀疏性控制更好 |
| B1-residual | 残差 SAE（多级分解） | 待验证 | 第一级粗分解 + 第二级残差修正 |
| B1-constrained | 结构约束 SAE（编码器稀疏/低秩/分块对角） | 待验证 | 训练时约束编码器结构，联合解决 B 和 C1 |
| B2 | 字典学习（K-SVD 等） | 未探索 | 经典稀疏表示方法 |
| B3 | 混合方法 | 未探索 | |

**B 的关键参数**：N（基向量库大小，当前 8h）、K（激活数，当前 ~128）

### C1: 怎么选择激活哪些基向量？

从 N 个候选中选出 K 个。当前核心瓶颈。

| ID | 方案 | 状态 | 访存量 | 备注 |
|----|------|------|--------|------|
| C1a | SAE 编码器直接计算 | 不可行 | h × 8h = 8h² | 比原始 matmul 还大 |
| C1b | ANNS (top-K MIPS) | 不可行 | 随 K 增长 | K=128 时无优势 |
| C1c | 分组路由（聚类 + 小型路由器） | 待验证 | h×G + 组内搜索 | G 为组数 |
| C1d | 哈希路由（LSH 映射到预定义集合） | 待验证 | 极低 | 精度损失未知 |
| C1e | token 间增量更新 | 待验证 | 取决于重叠率 | 需先验证重叠率假设 |
| C1f | 编码器低秩近似（E ≈ UV） | 待验证 | h×r + r×8h | 取决于有效秩 r |
| C1g | 稀疏编码器 | 待验证 | 远 < 8h² | 需在 B1-constrained 中联合训练 |

### C2: 怎么确定系数？

选定 K 个基向量后，如何计算组合系数 α_i。

| ID | 方案 | 状态 | 开销 | 备注 |
|----|------|------|------|------|
| C2a | 编码器内积 α_i = x · E_{:,i} | ← 当前 | O(K × h) | 简单快速，但系数未必最优 |
| C2b | 完整最小二乘 | 不可行(K=128) | O(K²h + K³) | K=128 时 compute-bound，19x 原始 matmul |
| C2c | CG 迭代求解（t=10 次） | 已验证：精度有效，性价比待定 | O(tKh) 计算, Kh 访存 | MSE↓10~17%，但 p 仅↓1~3pp，额外计算可能抵消收益。详见 [ideas/cg-coefficients.md](ideas/cg-coefficients.md)，[实验](experiments/20260315-cg-coefficients-phase1.md) |
| C2d | 其他优化方法 | 未探索 | | |

### D: 怎么保证精度？

处理分解引入的近似误差。

| ID | 方案 | 状态 | 访存量 | 备注 |
|----|------|------|--------|------|
| D1 | 逐维在线精确补偿 | ← 当前 | p × h × n | 每个补偿维度访问 W 的一列 |
| D1-approx | 逐维近似补偿（低秩列近似） | 待验证 | < p × h × n | 用列的低秩近似替代精确列 |
| D1-batch | 低秩批量残差修正 | 待验证 | r × (h + n) | 如果残差低秩则远小于 D1 |
| D2 | 模型微调吸收误差（类似 QAT） | 待验证 | 0（推理时无额外开销） | 训练成本高，但可能消除补偿 |
| D3 | 不补偿，接受误差 | 待验证 | 0 | 需要 B 的精度足够高 |

**D 的关键参数**：τ（缩放系数，当前 0.5，目标 0.2~0.3）、p（补偿比例，当前 ~20%@τ=0.5）

### D1-sub: 阈值机制（D1 的子决策）

仅在选择 D1 系列时适用。

| ID | 方案 | 状态 | 备注 |
|----|------|------|------|
| D1s-current | θ(Kneedle) × τ 全局统一缩放 | ← 当前 | τ=0.5 时 p≈20% |
| D1s-adaptive | 每算子独立 τ | 待验证 | 在总精度预算内最大化加速 |
| D1s-relative | 相对误差阈值 | 待验证 | 可能比绝对误差更稳定 |

## 4. 耦合关系矩阵

### 4.1 "如果改变 X，对 Y 有什么影响"

读法：行是"你改变的决策"，列是"受影响的决策"。

| 改变 ↓ \ 影响 → | B (基向量库) | C1 (选择算法) | C2 (系数) | D (精度保证) |
|-----------------|-------------|---------------|-----------|-------------|
| **B: N 变小** | 库更小 | 搜索空间↓，C1a/C1b 可能重新可行 | 不变 | 精度↓，p↑ |
| **B: N 变大** | 库更大 | 搜索空间↑，C1 更难 | 不变 | 精度↑，p↓ |
| **B: 换 Gated SAE** | 精度可能↑ | 门控网络可作为 C1 的轻量路由 | 不变 | p↓ |
| **B: 约束编码器结构** | 训练受约束，精度可能↓ | 编码器可直接用于 C1 | 不变 | 间接影响 |
| **C1: 选择质量↓** | 不变 | - | 系数可能变差 | p↑ |
| **C2: 换最小二乘** | 不变 | 不变 | - | 系数更优，p↓ |
| **D: 换成 D2(微调)** | 精度要求↓，N 可↓ | N↓ → 搜索空间↓ | K 可能可↓ | p→0 |
| **D: 换成 D1-batch** | 不变 | 不变 | 不变 | 补偿成本结构改变 |
| **A2: 换条件分解** | 每个子库更小更专精 | 两步选择：选簇+簇内选 | 不变 | 精度↑，p↓ |

### 4.2 关键耦合路径

以下是影响链条最长、杠杆效应最大的几条耦合路径：

**路径 1：D2(微调) → B(N↓) → C1(搜索空间↓)**
```
选择 D2 消除补偿 → B 的精度要求降低 → N 可以缩小 → C1 搜索空间缩小 → 之前不可行的 C1a/C1b 可能重新可行
杠杆效应：改变 D 一个点，解锁 B 和 C1 两个瓶颈
```

**路径 2：B1-constrained(结构约束SAE) → C1(编码器可直接用) + D(精度可能变化)**
```
训练时约束编码器为稀疏/分块对角 → 编码器在线计算开销可接受 → C1 直接用编码器 → 但精度可能受损
杠杆效应：B 的一个选择同时决定 C1 是否可行
```

**路径 3：A2b(条件分解) → B(子库更小) + C1(搜索空间↓) + D(精度↑)**
```
将输入空间分簇 → 每个簇有专精的小基向量库 → 库更小所以选择更快 → 更专精所以精度更高 → p 更低
杠杆效应：改变 A2 一个点，B/C1/D 全部受益
```

**路径 4：C2c(CG系数) → B(基向量库质量↑) + C1(选择要求↓) + D(p↓)**
```
推理用 CG → 系数更优 → p↓
训练也用 CG → 选择与系数解耦 → 基向量库质量↑ → p 进一步↓
                             → 编码器职责简化 → C1 可用更轻量方法
                             → K 的下界可能降低 → 查表访存↓ + ANNS 可能重新可行
杠杆效应：C2 一个改变，同时影响 B、C1、D 三个分支
详见：ideas/cg-coefficients.md
```

## 5. 使用说明

### 实验前
1. 确定要改变的节点（比如 C2a → C2b）
2. 查耦合矩阵 §4.1，看会影响哪些其他节点
3. 在 `experiments/` 下创建实验文档（按 [模板](experiments/_template.md)），把被影响的节点纳入观测指标
4. 切换到独立 worktree 执行实验

### 实验后
1. 更新对应节点的状态（当前/已否定/待验证）
2. 更新 `experiments/` 中对应实验文档的 §7（实际结果）和 §8（结论），状态改为 completed / abandoned
3. 在 `research-log.md` §6 添加一句话实验记录，附链接到实验文档
4. 检查耦合矩阵：实验结论是否改变了其他节点的可行性？
5. 更新"当前路径"

### 新增决策点
1. 在对应的决策分支下添加新行
2. 更新耦合矩阵（新方案与其他分支的关系）
3. 在 `research-log.md` §5 添加对应的开放问题
